以下の手順で、ローカルの現在のブランチと$ARGUMENTSブランチの差分に対してコードレビューを行ってください。

1. 現在のブランチの確認:

`git branch --show-current` を実行して、
現在チェックアウトしているブランチを確認してください。

2. 差分の取得:

`git diff $ARGUMENTS...HEAD` コマンドを使用して、
$ARGUMENTSブランチから現在のブランチへの差分
（コミット済み）を取得してください。
未コミットの変更やUntrackedなファイルは対象外とします。

3. レビューの実施:

取得した差分に基づき、コードレビューを行ってください。
- レビュー観点は、プロジェクトルートの `CLAUDE.md`
  および関連するスキル（有効化されている場合）に
  従ってください。
- 特に、セキュリティ、バグの可能性、可読性、保守性、
  およびコーディング規約への準拠に注意してください。
